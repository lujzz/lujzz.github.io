数据结构整理

数据结构整理

Chapter1：基本概念

  1. 数据结构：是相关之间存在一种或者多种特定关系的数据元素的集合。

  2. 逻辑结构：是指数据元素之间的逻辑关系。

    ​ 分类：集合，线性结构，树形结构，图状结构。

  3. 物理结构：数据结构在计算机中的表示（映像），依赖于计算机语言。

    ​ 分类：

    ​ 顺序结构（物理位置相邻）；

    ​ 链接存储（记录相邻元素的物理位置）；

    ​ 索引存储（类似于目录）；

    ​ 散列存储（通过关键字直接计算出元素的物理地址）。

  4. 算法的5个特征：有穷性，确定性，可行性，输入，输出。

  5. 算法的复杂度：时间复杂度、空间复杂度。

Chapter2：线性表

  1. 逻辑结构：n个相同数据类型的元素的有限序列。（一对一）

    ​ 表头元素，表尾元素，直接前驱，直接后继。

  2. 顺序表：线性表的顺序存储，使用地址连续的存储单元——存储密度高。

    • 逻辑相邻——物理位置相邻——可以通过首地址和元素序号找到指定元素。

    • 建立顺序表的三个属性：初始位置，最大存储容量，当前长度。

    • 动态分配空间。

    • 插入：

      • 判断i的值是否正确——判断表长是否超过数据长度——元素后移（从后往前移动）——插入特定元素，修改表长。
      • 时间复杂度：O(n)

    • 删除同上。

  3. 链表：存储相邻元素的物理位置，物理位置不相邻。（头结点）

    *在任何运算之前都应该考虑该操作的合理性！

    • 插入：m,n之间插入k
      $$
      k\rightarrow next = m\rightarrow next \
      m\rightarrow next = k
      $$

    • 删除：m,k,n之间删除k
      $$
      k = m\rightarrow next \
      m\rightarrow next = k\rightarrow next \
      free(k) :释放内存
      $$

  • 双链表：同理，注意删除。在进行插入和删除时，可以进行简易画图hhh。

  • 循环链表：表中最后一个节点指向头节点。

  • 静态链表：用数组来描述线性表的链式存储结构。

    ​ 存储在数组中的序号

Chapter3：栈和队列

  1. 栈(Stack)

    只允许在一端进行操作的线性表。——栈顶、栈底。

    后进先出

    • 顺序表——操作：判断是否为空，进栈，出栈，读取栈顶元素。
    • 共享栈
    • 链式栈

  2. 队列

    只允许在一端进行插入，而在另一端进行删除的线性表。——队头Front，队尾Rear

    先进先出

    • 顺序队列

    • 循环队列——首尾相连——操作：入队、出队。

      判断队列的空/满

    • 链式队列

    • 双端队列

  3. 栈的应用——括号匹配、表达式求值（后缀表达式）、递归

    ​ *后缀表达式：从左到右遍历中缀表达式，如果是数字则输出，符号和栈顶元素比较优先级，小于等于则进栈，大于则出栈直到出现优先级更大的栈顶元素，若是左括号则直接进栈，若是右括号，则出栈直到左括号。

    ​ 或：先按运算符优先级对所有运算符和它的运算数加括号(原本的括号不用加)，再把把运算符移到对应的括号后，最后去掉括号即可。

Chapter4：树

  1. 基本概念：递归定义的结构。

    ——节点、根节点、节点的度、节点关系、层次、深度

    ——性质：

    • 树中的结点数等于所有结点树加一。除根节点外，一个度可以看作是指对于一个结点的指向，所以结点数等于度的个数加一。
    • 度为m的树中第i层至多有m^(i-1) 个结点。（数学归纳法）
    • 高度为h的m叉树至多有(m^h-1)/(m-1)个结点
    • 具有n个结点的m叉树的最小高度为logm(n(m-1)+1)

  2. 存储结构：

    • 顺序存储结构：双亲表示法——用一组连续的存储空间存储树的结点，同时在每个结点中，用一个变量存储该结点的双亲结点在数组中的位置。
    • 链式存储结构：
      • 孩子表示法
      • 孩子兄弟表示法

  3. 二叉树：

    • 二叉树——左子树、右子树——二叉树。

    • 存储结构：

      • 顺序存储：用一组地址连续的存储单元依次自上而下、自左至右存储完全二叉树上的结点元素。
      • 链式存储：在结点结构里考虑两个指针指向该结点的两个孩子。

    • 遍历：

      • 先序遍历：根节点——左子树——右子树
      • 中序遍历：左子树——根节点——右子树
      • 后序遍历：左子树——右子树——根节点

    • 线索二叉树：使用空指针存储前驱和后继元素

    • 哈夫曼树：带权路径长度最小——由叶子结点由下到上根据权重提取树的结构。——代码

Chapter5：图

  1. 基本概念：顶点、边、有向图、无向图、简单图、子图、连通图、度······

  2. 存储结构：邻接矩阵（顺序存储）、邻接表（链式存储）——十字链表、邻接多重表。

  3. 遍历：

    • 深度优先搜索（DFS）：递归，栈
    • 广度优先搜索（BFS）：遍历，队列

  4. 应用

    • 最小生成树

      • Prim ：由顶点寻找权值最小的边，适合稠密图
      • Kruskal：由权值最小的边确定顶点，适合稀疏图

    • 最短路径

      • Dijkstra算法（一个源点到其余顶点的最短路径）（两个数组）

        初始——确定一个顶点——更新——重复

      • Floyd算法 （任意两点之间的最短路径）

Chapter7：排序

*稳定性

  1. 插入类：（比较+移动）

    • 直接插入排序 O(n^2) ：待插入的元素—在已经排序的元素中寻找待插入元素的位置—移动已排序元素，插入待插入的元素—重复

      ——从后往前，设置哨兵

    • 折半插入排序 O(n^2)

    • 表插入排序

    • 希尔排序：将原序列分为若干子序列，逐渐减小增量直到最后减为1

  2. 交换类：

    • 冒泡排序：两两比较

    • 快速排序 O(nlogn) ：枢轴记录

      ​ 借助栈进行操作，空间复杂度 O(n)

  3. 选择类：

    • 简单选择排序
    • 锦标赛排序：时间复杂度：O(nlogn) 空间复杂度：O(n)
    • 堆排序：时间复杂度：O(nlogn)
      • 堆：堆是一棵完全二叉树，而且满足任何一个非叶结点的值都不大于(或不小于)其左右孩子结点的值。
      • 建堆：初始建堆，然后从第n/2个元素向前进行调整。
      • 获取堆顶元素后调整堆：将最后一个元素放在堆顶，然后进行交换调整。

  4. 归并类：

    • 归并排序：时间复杂度：O(nlogn)
