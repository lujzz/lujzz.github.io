kitti数据集使用：

  • {0,1,2,3}是相机索引，其中0代表左灰度，1代表右灰度，2代表左彩色，3代表右边彩色相机

  • 数据集内容：

    • calib：校准数据，txt

      • P0,P1,P2,P3：12个数
      • R0_rect：9个数
      • Tr_velo_to_cam：12个数

    • image_2：图像数据，png

    • label_2：标签，txt

      • ​ 15个数据

      • camera frame下的标签dimensions和location转化为顶点坐标，再通过P2矩阵转化为image frame下的 2D 框后，可以与标签bbox基本重合

    • velodyne：点云数据，bin

  • 坐标系准换：

    坐标系分别有velodyne frame, camera frame, image frame
    涉及3个转换矩阵，分别是P2，R0_rect，Tr_velo_to_cam

    P2有12个数，reshape为（3，4）

    R0_rect有9个数，reshape为（3，3），补0扩展为（4，4），再把右下角的那个元素置为1

    Tr_velo_to_cam有12个数，reshape为（3，4），补0扩展为（4，4），再把右下角的那个元素置为1

    • y--velodyne frame--(x,y,z,1)'

      Tr_velo_to_cam * y--camera frame(0)

      R0_rect * Tr_velo_to_cam * y--camera frame(2)--(x,y,z,1)

      P2 * R0_rect * Tr_velo_to_cam * y--image frmae(2)--(u,v,w)注意此时得到的 u,v 并非是像素坐标，需要除以 w 才是最后的像素图像坐标，那么投影前的 3D object 有8个点，投影到image frame上也有8个（u, v）点，如何得到对应的 2D 检测框呢？取8个点（u, v）的最大值最小值就是 2D 检测框的对角坐标
